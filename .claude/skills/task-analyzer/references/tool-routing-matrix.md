# 도구 라우팅 매트릭스

> 작업 유형별로 어떤 MCP 도구를 어떤 순서로 호출해야 하는지 가이드.
> task-analyzer가 Phase 3(실행 전략 선택)에서 참조한다.

---

## 1. 정보 조회 작업

**"~에 대해 알려줘", "지난번에 뭐라고 했지", "기존 데이터 확인"**

| 순서 | 도구 | 파라미터 | 목적 |
|------|------|----------|------|
| 1 | `memory_search` | query, tag, category | 내부 메모리에서 관련 정보 검색 |
| 2 | `memory_get` | id 또는 path | 검색 결과 중 상세 내용 조회 |
| 3 (필요 시) | `web_search` | 검색 쿼리 | 메모리에 없는 정보는 웹에서 |

**판단 기준**: 항상 `memory_search` 먼저. 결과가 부족하면 `web_search`.

---

## 2. 정보 저장 작업

**"기억해", "저장해", "기록해", "잊지마"**

| 순서 | 도구 | 파라미터 | 목적 |
|------|------|----------|------|
| 1 | `memory_search` | query (중복 확인) | 동일 정보 이미 저장 여부 확인 |
| 2a (신규) | `memory_store` | category, title, content, tags, importance | 새 메모리 생성 |
| 2b (갱신) | `memory_update` | id, mode:"append", content | 기존 메모리에 추가 |

**카테고리 선택 기준**:
- 프로젝트 관련 → `projects` (subcategory: 프로젝트명)
- 사람 관련 → `people`
- 기술/지식 → `knowledge` (subcategory: 분야)
- 할일/계획 → `tasks`
- 분류 불명 → `inbox` (나중에 정리)

## 3. 웹 리서치 작업

**"조사해줘", "검색해줘", "최신 정보 확인"**

| 순서 | 도구 | 파라미터 | 목적 |
|------|------|----------|------|
| 1 | `memory_search` | query | 기존 조사 이력 확인 |
| 2 | `web_search` | 한국어 쿼리 | 한국어 소스 수집 |
| 3 | `web_search` | 영어 쿼리 | 영어 소스 수집 (범위 확대) |
| 4 | `web_fetch` | URL | 주요 페이지 상세 내용 수집 |
| 5 | `memory_store` | category, tags, importance | 수집 결과 영구 저장 |

**쿼리 설계 규칙**:
- 한국어 + 영어 이중 검색으로 커버리지 확대
- 시간 제한 키워드 추가 (예: "2026", "latest")
- 도메인 특화 키워드 포함 (예: Web3, smart contract)

## 6. 예약/자동화 작업

**cron 표현식 빈도 가이드**:
| 자연어 | cron |
|--------|------|
| 매일 아침 9시 | `0 9 * * *` |
| 평일 아침 9시 | `0 9 * * 1-5` |
| 매주 월요일 | `0 9 * * 1` |
| 매시간 | `0 * * * *` |
| 6시간마다 | `0 */6 * * *` |
| 매월 1일 | `0 9 1 * *` |

## 10. 복합 작업 도구 선택 우선순위

여러 도구가 후보일 때의 선택 기준:

| 우선순위 | 규칙 |
|----------|------|
| 1 | `memory_search` — 항상 먼저 (기존 컨텍스트 활용) |
| 2 | 특화 MCP 도구 — 해당 작업 전용 도구가 있으면 사용 |
| 3 | `web_search` / `web_fetch` — 내부에 없는 정보만 외부에서 |
| 4 | 내장 파일 도구 — 문서/파일 생성·편집 |
| 5 | `bash` — 위 도구로 불가능한 경우에만 (서브에이전트 위임 권장) |

**모든 작업 종료 시 공통**:
- `memory_daily_log` — 실행 로그 기록
- `memory_store` — 중요 결과물 영구 저장 (importance 5 이상)
- `messenger_send` — 사용자가 알림을 요청했거나 중요도 8 이상인 결과
